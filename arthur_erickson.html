Arthur Erickson Website

This is the Arthur Erickson website project from IAT 235 Information Design created by Jeffrey Cheng.

Project Description

The Arthur Erickson website was a deep dive into showcasing the legacy of a visionary architect. The project began with research into Erickson's work to understand the philosophy behind his designs. The website was envisioned as a digital exhibition, highlighting Erickson's projects with rich visual and textual narratives.

This was a school project from IAT 235 which was Information design from SFU. I was tasked to create a website from a famous architect. I chose Arthur Erickson because I not only was interested in his work, but I also know that given the choice between creating a website for a company or Arthur Erickson, I had a greater desire to create a website for Arthur Erickson because at that time I did not have much experience in creating websites for companies. I added the information about the introduction of the architect, the projects he has done, and the philosophy behind his designs using HTML/CSS.

Introductory section of the website, showcasing the minimalist visual language inspired by Erickson’s architectural philosophy. The design uses clean typography and balanced layout to reflect the spatial harmony found in Erickson’s work.

Process Analysis Part 1

Developing the site called for a minimalist design that could mirror Erickson's architectural style—modern and nature-centric using Figma. With this in mind, I employed a clean layout with plenty of whitespace, allowing Erickson's work to stand front and centre. Interactive galleries were developed using HTML5 and CSS3, echoing the sophistication of Erickson's designs.

Screenshot of the homepage layout with interactive elements. This demonstrates the use of whitespace and structure to guide the viewer’s attention—echoing Erickson’s approach to spatial design.

Process Analysis Part 2

I first started by adding the information about the introduction of the architect, the projects he has done, and the philosophy behind his designs using Figma. I also added the navigation bar to the top of the page so that the user can navigate through the website easily. I also added contact information, date and location as well as admission fee and ticket information of the website all under a single template.

Early development stage of the website in Figma. This image shows placeholder content being replaced with real exhibition data, emphasizing the integration of biographical, project, and ticketing information within a unified template.

Problem 1: Information Overload and Layout Confusion

Initially, all the content about Arthur Erickson – including his biography, architectural projects, and design philosophy – was presented on a single page. This design approach resulted in a cluttered and confusing user interface, as users typically expect different types of content to be segmented across various pages.

Draft version of the homepage before content reorganization. This version reveals how cramming all information on a single page created cognitive overload for users, making navigation difficult.

Solution:

The solution involved reorganizing the website’s content into separate pages. By segmenting the introduction, detailed project information, event details, and contact information onto different pages, the website's navigation became clearer and more user-friendly. This approach aligned with best practices for web design, enhancing the overall usability and aesthetic of the site.

Revised structure with content distributed across multiple pages, improving clarity and reducing user confusion. The redesign follows common UX principles to enhance readability and flow.

Problem 2: Need for a User-Focused and Minimalist Design

The challenge was to create a design that was reflective of Erickson's architectural style, which is modern and integrated with nature. The original design did not adequately meet these requirements, potentially failing to engage users effectively.

Initial site built on a single rigid template. The lack of variation between sections failed to reflect the diversity and elegance of Erickson’s portfolio.

Solution:

A minimalist design was adopted to mirror the elegance and simplicity of Erickson's architectural work. This was achieved by using a clean layout with ample whitespace and interactive galleries that allowed Erickson's projects to be the focal point. HTML5 and CSS3 were used to create sophisticated interactive elements, contributing to a seamless user experience.

Final layout implementing minimalist design principles. The use of grids, soft contrasts, and image-based galleries places Erickson’s architecture at the visual and conceptual forefront.

Reflection on Solutions:

Reflecting on the Arthur Erickson Architectural Portfolio project, it's clear that user experience is paramount. Shifting to a minimalist design enhanced the site's clarity and usability, while interactive elements invited deeper engagement with Erickson's work. The introduction of a clear navigation bar and the strategic distribution of content across multiple pages were critical in resolving initial user interface challenges. This project highlighted the importance of iterative design and user feedback. By continuously refining the website, we were able to create a more intuitive and educational platform. These changes not only improved navigation but also ensured that each of Erickson's projects was given the attention it deserved. Ultimately, this process underlined the significance of adaptability and user-focused design in creating compelling digital experiences.
