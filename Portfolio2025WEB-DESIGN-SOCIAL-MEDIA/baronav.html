BaroNav

A Mobile Computing Group Final Project by Jeffrey Cheng & Max Nielsen.

Project Description

BaroNav is a barometric pressure tracking app developed for IAT 359: Mobile Computing at Simon Fraser University (SFU). The app leverages smartphone barometer sensors and GPS data to provide personalized, location-based weather insights. Users can track real-time atmospheric pressure, analyze historical trends, and integrate external weather services for comprehensive forecasting. Our focus was to make barometric pressure a key factor in weather-related decision-making while ensuring accurate, data-driven, and visually intuitive insights.

Main Functionalities & Development

I was responsible for designing and implementing the core functionalities of the app, including: Mobile Application Development (Java, XML, Android Studio) Developed a fully functional interactive map, allowing users to save, edit, and navigate locations seamlessly. Designed a smooth and responsive UI layout to enhance location management and usability. Implemented data visualization tools to track atmospheric pressure trends. User Interface & Experience Design (Figma, Wireframing, UX/UI Design) Refined XML layouts for the home screen and map navigation, improving intuitiveness. Designed pin-based selection and manual location entry for accessibility. Developed a visually clear and engaging UI to make barometric pressure insights easy to understand.

These are the essential elements for the home page and weather data which are designed in figma and developed in XML, Java and Android Studio. 

Weather Data Integration Challenges

One of the biggest challenges was implementing the weather API, as its complex data structure required extensive processing. Developing a custom WeatherService was necessary to fetch, store, and dynamically update pressure and weather conditions efficiently.

This is the data that is being obtained to display the weather conditions. 

Easily parse the response and link data to UI widgets

To optimize performance, I structured API calls using OkHttpClient in Java for efficient data parsing. Then, I linked responses to UI widgets in real-time, ensuring accurate and smooth weather updates.

The data is being displayed on the home page after the data is obtained.

The goal was to make a map easy for users to add, edit and manage locations. My partner Max has polished the layout. 

Improved UI/UX for Navigation

I coded the XML to refine Add/Edit Location layouts, ensuring seamless interaction with pin-based selection and manual entry. My partner further refined the layout for enhanced usability.
