BaroNav

A Mobile Computing Final Project by Jeffrey Cheng & Max Nielsen.

Project Description

BaroNav is a mobile app developed for IAT 359: Mobile Computing at SFU. It leverages smartphone barometer sensors and GPS data to provide personalized, location-based weather insights. Users can track real-time atmospheric pressure, analyze historical trends, and integrate external weather APIs for forecasting. Our focus was to make barometric pressure a meaningful factor in weather decision-making, with a data-driven and visually intuitive UI.

Main Functionalities & Development

I was responsible for designing and implementing key features, including: - Mobile Development (Java, XML, Android Studio) → Built an interactive map for saving, editing, and navigating locations. - UI Layout Design → Created responsive layouts to improve location management and usability. - Data Visualization Tools → Enabled users to view and track atmospheric pressure trends. - UI/UX Design (Figma, Wireframing) → Designed intuitive home and map screens, including pin-based selection and manual entry. Overall, I ensured the UI remained clear, accessible, and engaging.

Essential UI wireframes designed in Figma and implemented in XML/Java.

Weather Data Integration Challenges

A major challenge was processing the complex API data structure from external weather services. We had to build a custom WeatherService to fetch, store, and dynamically update pressure and weather conditions efficiently.

Weather data being retrieved and structured for real-time updates.

Easily Parse & Link Data to UI

To optimize performance, I structured API calls using OkHttpClient in Java for efficient parsing. Then, I mapped responses to UI widgets in real-time, ensuring smooth and accurate weather updates directly on the home screen.

Dynamic data-driven UI showing weather conditions updated in real-time.

Map Navigation Complexity

Designing an intuitive map system for adding, editing, and managing locations was a challenge. The XML layout required refinement to maintain responsiveness and smooth interaction.

Map interface allowing users to add, edit, and manage locations.

Improved UI/UX for Navigation

I refined the XML layouts to streamline map navigation with pin-based selection and manual entry options. My partner polished the visuals further, resulting in improved usability and smoother interactions.

Final XML layout for Add/Edit functionality with improved usability.

Reflection on Solutions:

BaroNav reinforced the importance of integrating complex APIs with efficient data handling and linking them to a user-friendly UI. The project improved my skills in Java, XML, API integration, and UX design, while also teaching me how collaboration strengthens problem-solving in technical projects.
